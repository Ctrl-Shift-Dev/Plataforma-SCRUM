Product Backlog

Conceitos

O Sprint Backlog é uma lista de tarefas específicas e detalhadas que a equipe de desenvolvimento seleciona a partir do Product Backlog para serem realizadas durante uma Sprint. Ele representa o trabalho concreto e detalhado que a equipe compromete-se a concluir durante o Período da Sprint.

Exemplos Práticos

Imagine que uma equipe de desenvolvimento esteja trabalhando em um projeto de desenvolvimento de software usando Scrum.

Aqui estão alguns exemplos práticos do Sprint Backlog:

Desenvolvimento de Recursos de Login: Um item do Product Backlog é "Implementar um sistema de login seguro". Para a Sprint atual, a equipe detalha as tarefas necessárias, como "Projetar a interface de login", "Desenvolver a funcionalidade de validação de senha" e "Configurar a autenticação de dois fatores". Essas tarefas fazem parte do Sprint Backlog.

Correção de Bugs Prioritários:A equipe recebe uma solicitação urgente para corrigir três bugs críticos relatados pelos usuários. Eles identificam esses bugs como itens do Sprint Backlog para uma Sprint específica, detalhando as etapas necessárias para investigar e resolver cada problema.

Desenvolvimento de uma nova funcionalidade:Um novo recurso é solicitado pelo Product Owner. A equipe desdobra esse pedido em tarefas específicas, como "Realizar pesquisa de mercado para entender os requisitos do novo recurso", "Projeto de interface de usuário" e "Desenvolvimento do código". Todas essas tarefas se tornam parte do Sprint Backlog
